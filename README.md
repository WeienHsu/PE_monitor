# PE Monitor 📊

每日監測自選美股的本益比（P/E）位置，整合即時新聞情緒分析，幫助判斷進出場時機。

系統會自動分析每檔股票是否適合使用 P/E 或 P/B 區間法，根據歷史百分位輸出估值訊號，並結合最新新聞情緒產生複合訊號。

---

## 功能亮點

- **估值訊號**：根據 5 年歷史 P/E 百分位輸出 5 級訊號（BUY → SELL）
- **新聞情緒分析**：抓取 Finnhub / Yahoo Finance RSS 新聞，用 VADER 離線評分
- **複合訊號**：估值 × 情緒 = 15 格矩陣（🌟 強力買進 → 🚨 強力賣出）
- **P/E Band 走勢圖**：歷史每日 P/E 曲線 + 百分位色帶 + 雙軸股價疊圖
- **股票適合度自動分析**：穩定型 ✅ / 成長型 🟡 / 景氣循環型 ❌ / ETF 📦
- **持倉管理**：追蹤成本、即時損益 %
- **本地快取**：股價 / 財報 / 新聞分別快取，避免重複 API 呼叫
- **Streamlit 介面**：互動式 Web UI，三個頁面

## 快速開始

### 1. 安裝

```bash
git clone https://github.com/WeienHsu/PE_monitor.git
cd PE_monitor

python -m venv .venv
source .venv/bin/activate      # macOS/Linux
# .venv\Scripts\activate       # Windows

pip install -r requirements.txt
python -m nltk.downloader vader_lexicon   # 下載 VADER 情緒模型（首次需要）
```

### 2. 初始設定（選用）

```bash
cp .env.example .env
```

編輯 `.env`：

```dotenv
# 自選股（逗號分隔）
WATCHLIST=GOOGL,AAPL,MU,NVDA

# 持倉（ticker:成本:股數，沒有可留空）
HOLDINGS=GOOGL:175.0:10,AAPL:195.0:5

# Finnhub API Key（免費申請：https://finnhub.io/register）
FINNHUB_API_KEY=your_key_here

# 其他設定（有預設值，可不填）
PE_HISTORY_YEARS=5
ENTRY_PERCENTILE=25
EXIT_PERCENTILE=75
```

> `.env` 只在**第一次啟動**時用來初始化 `config.json`，之後請透過 Streamlit 介面操作。

### 3. 啟動 Streamlit

```bash
.venv/bin/streamlit run app.py
```

開啟瀏覽器 → `http://localhost:8501`

## 訊號說明

### 如何使用這個系統

**日常操作只需要看「綜合訊號」即可。** 系統已自動將股票類型、補充估值指標、新聞情緒全部整合進最終訊號。

| 綜合訊號 | 建議行動 |
|---------|---------|
| 🌟 強力買進 | 估值歷史低位 + 正面消息，積極考慮建倉 |
| 🟢 買進 | 估值偏低，可分批進場 |
| 🟢⚠️ 謹慎買進 | 估值低但有負面消息，少量試單 |
| 🔵 觀察 | 偏低但不急，持續追蹤 |
| ⚪ 中性 | 合理估值，不建議主動進出 |
| 🟡 謹慎 | 偏貴，避免加碼 |
| 🔴 賣出 | 歷史高估，考慮減倉或停利 |
| 🔴⚠️ 謹慎賣出 | 高估但有正面消息支撐，小量減碼 |
| 🚨 強力賣出 | 極度高估 + 負面消息，盡快離場 |

> **訊號不是交易指令**，只是輔助判斷。倉位控制、市場環境仍需自行評估。

---

### 何時需要懷疑訊號？

綜合訊號在以下情況可能失真，需要進一步查看個股展開的補充指標：

| 情況 | 問題所在 | 如何處理 |
|------|---------|---------|
| 股票類型為「景氣循環型 ❌」 | 景氣高峰時 EPS 虛高，P/E 看起來便宜但其實高估 | 以 **EV/EBITDA < 8** 才算真低估 |
| 顯示 ⚠️ EPS 過期（> 100 天） | 財報資料陳舊，P/E 可能不準 | 等最新財報後再決策 |
| 股票剛上市或歷史不足 5 年 | 百分位樣本太少，參考價值低 | 多看補充指標，減少倉位 |
| 新聞來源顯示「使用 RSS 備用」 | 情緒分析品質降低 | 手動查閱近期新聞確認 |

---

### 估值訊號（5 級，系統內部計算用）

| 訊號 | 條件 | 說明 |
|------|------|------|
| 🟢 BUY | P/E < 25th 百分位 | 歷史低估區間，考慮進場 |
| 🔵 WATCH | 25th–35th 百分位 | 接近進場區，觀察 |
| ⚪ NEUTRAL | 35th–65th 百分位 | 合理估值 |
| 🟡 CAUTION | 65th–75th 百分位 | 接近出場區，謹慎 |
| 🔴 SELL | P/E > 75th 百分位 | 歷史高估區間，考慮減倉 |

> 百分位門檻可在 Sidebar「設定」中調整。

### 新聞情緒（3 級）

| 標籤 | 說明 |
|------|------|
| 📰🟢 正面 | 近期新聞以利多為主 |
| 📰⚪ 中性 | 無明顯情緒偏向 |
| 📰🔴 負面 | 近期新聞以利空為主 |

VADER 對每篇文章的標題（70%）+ 摘要（30%）評分，並對近期文章給予更高時間權重（指數衰減）。

### 複合訊號矩陣（估值 × 情緒）

|  | 正面新聞 | 中性新聞 | 負面新聞 |
|--|---------|---------|---------|
| **BUY** | 🌟 強力買進 | 🟢 買進 | 🟢⚠️ 謹慎買進 |
| **WATCH** | 🟢 買進 | 🔵 觀察 | ⚪ 中性 |
| **NEUTRAL** | 🔵 觀察 | ⚪ 中性 | 🟡 謹慎 |
| **CAUTION** | ⚪ 中性 | 🟡 謹慎 | 🔴 賣出 |
| **SELL** | 🔴⚠️ 謹慎賣出 | 🔴 賣出 | 🚨 強力賣出 |

> **設計原則**：新聞情緒最多讓訊號偏移一級，不會整個翻轉。估值是主訊號，新聞是修正因子。

## 股票類型與補充指標

### 股票類型分類（多因子自動判斷）

系統根據 **4 個維度的積分制評分**自動將股票分類，影響哪些補充指標會被納入綜合訊號：

| 因子 | 偏向成長型 | 偏向穩定型 | 偏向循環型 |
|------|-----------|-----------|-----------|
| EPS 波動率 | 30–60% (+2) | < 30% (+2) | > 60% 或有負值 (+2) |
| 營收成長率 | > 20% (+2)；10–20% (+1) | < 2% (+1) | < 0% (+1) |
| Beta | 1.2–1.5 (+1) | < 0.8 (+1) | > 1.5 (+1) |
| 股息殖利率 | — | ≥ 2% (+1) | — |

最高積分者為分類結果；同分時：穩定型 > 成長型 > 循環型（保守原則）。分類原因欄位會顯示各因子數值供確認。

| 類型 | 評分最高條件 | 關鍵指標 | 備註 |
|------|------------|---------|------|
| **穩定型** ★★★★★ | 低 EPS 波動 + 低 Beta + 有配息 | P/CF（×2）、PEG、Forward P/E、股息殖利率 | 現金流與配息能力最重要 |
| **成長型** ★★★ | 高營收成長 + 中高 Beta | P/S（×1）、PEG（×2）、Forward P/E（×2）、營收成長率 | P/CF 停用（成長股大量再投資） |
| **景氣循環型** ★★ | EPS 曾負值或高波動 | P/CF、EV/EBITDA | PEG/Forward P/E 不可信；訊號調整幅度保守（±1 級） |
| **ETF** ★★★ | 無 EPS | 僅 Forward P/E | 估值訊號顯示 N/A |

> 股票類型可在「自選股管理」頁面手動覆寫，系統標示為「✏️ 手動」vs「🤖 自動」。

### 補充估值指標快速對照

這些指標顯示在個股展開區，用來驗證綜合訊號的可信度：

| 指標 | 公式 | 低估 | 合理 | 高估 | 適用類型 |
|------|------|------|------|------|---------|
| **P/CF** | 股價 ÷ 每股營業現金流 | < 10 | 10–20 | > 20 | 穩定、循環型 |
| **PEG** | P/E ÷ EPS 成長率 % | < 1.0 | 1–2 | > 2.0 | 穩定、成長型 |
| **Forward P/E** | 股價 ÷ 預估未來 12 月 EPS | Forward < Trailing × 0.9 ✅ | 持平 | Forward > Trailing × 1.1 ⚠️ | 所有類型 |
| **EV/EBITDA** | 企業價值 ÷ EBITDA | < 8 | 8–15 | > 15 | 景氣循環型 |
| **P/S** | 股價 ÷ 每股年營收 | < 4 | 4–20 | > 20 | 成長型（獲利不穩定時）|
| **營收成長率** | 年營收年增率 | > 15% | 5–15% | < 5% | 成長型（確認成長論點）|
| **股息殖利率** | 年化股息 ÷ 股價 | ≥ 2% | 0.5–2% | < 0.5% | 穩定型 |
| **Beta** | 相對大盤波動性 | < 0.8（防禦型）| 0.8–1.2 | > 1.5（高波動）| 所有類型（分類參考）|

> **P/S 注意事項**：業界差異大（SaaS 均值約 10x，消費品約 2x），閾值採保守設定，請搭配產業背景解讀。

> **PEG < 1 特別重要**：代表股票成長速度比股價上漲更快，即使 P/E 看起來高也可能是便宜的。

## Strategy D 技術訊號

Strategy D 是「技術面反轉」的確認訊號，在儀表板圖表上顯示為**紫色三角形**。

**觸發條件（兩者必須同時成立）：**

1. **KD 超賣金叉**：K 值從 K < 20（超賣區）向上穿越 D 值
2. **MACD 柱狀圖收斂**：最近 3 根柱狀圖皆為負值，但逐漸靠近零軸（空頭動能減弱）

**如何解讀：**
- Strategy D 是「+1 票」的加分項目，不單獨決定買賣
- 最佳情境：估值訊號已在 BUY/WATCH 區，**再加上 Strategy D 觸發 → 進場信心加強**
- Strategy D 單獨觸發但估值偏高 → 不建議買進

**參數設定（設定頁面可調）：**

| 參數 | 預設值 | 說明 |
|------|--------|------|
| KD 回看窗口 | 10 根 | 金叉最多幾根 K 棒前發生 |
| MACD 收斂根數 | 3 根 | 需要連續幾根柱狀圖收斂 |
| 空頭恢復門檻 | 70% | MACD 從低點反彈幾 % 算有效 |
| KD 超賣門檻 | 20 | K 值低於此才算超賣 |

## 新聞情緒設定

### 設定 Finnhub API Key（建議）

**方法一：直接編輯 `config.json`**

```json
"settings": {
    "finnhub_api_key": "你的key填這裡",
    ...
}
```

**方法二：`.env` 檔案（Key 不進 git）**

```bash
echo 'FINNHUB_API_KEY=你的key填這裡' > .env
# 刪除 config.json 後重啟 Streamlit（系統會自動重新生成）
```

免費申請：[finnhub.io/register](https://finnhub.io/register)（60 次/分鐘，1 年歷史）

### Sidebar 狀態指示

| 狀態 | 顯示 |
|------|------|
| Finnhub 正常 | `✅ Finnhub 正常運作` |
| 未設定 Key，使用 RSS | `⚠️ 使用 RSS 備用（建議設定 Finnhub Key）` |
| 達速率上限，切換 RSS | `⚠️ Finnhub 達速率上限，已切換 RSS` |
| API Key 無效 | `🔴 Finnhub API Key 無效，請重新設定` |
| 所有來源失敗 | `🔴 無法取得新聞資料` |

### 降級策略

1. **Finnhub**（主力）：需免費 API Key，60 req/min，1 年歷史
2. **Yahoo Finance RSS**（備用）：無需 Key，穩定性較低，自動啟用
3. **無資料**：`available=False`，複合訊號直接沿用估值訊號，不報錯

### 新聞相關性過濾

系統會自動過濾與個股無關的市場整體新聞（Fed 升息、通膨、大盤走勢等），只保留包含股票代號或公司名稱關鍵字的文章，避免大盤消息稀釋個股情緒評分。過濾邏輯：

- 萃取公司名稱中長度 ≥ 5 字母的詞彙作為比對 token
- 文章超過 5 篇時才啟用過濾（避免過濾後剩 0 篇的邊界情況）

### 中文標題處理

VADER 情緒模型為純英文詞彙庫，對中文標題會產生接近 0 的無效評分。系統已自動偵測包含中文字元（CJK）的標題並跳過評分，將其視為中性（score = 0），避免誤導情緒結果。

## P/E 計算原理

```
TTM EPS = 最近 4 季 淨利 ÷ 稀釋股數（四季加總）
日 P/E  = 當日收盤價 ÷ TTM EPS
```

- EPS 在每次財報公布後更新，兩次財報之間為固定值
- 距上次財報超過 100 天時顯示 ⚠️ 過期警告
- EPS 為負時自動切換為 P/B 顯示
- ETF（如 GLD、SOXX）無 EPS，估值訊號顯示 N/A，但仍會抓取並顯示新聞情緒

## 注意事項

- 資料來源為 [yfinance](https://github.com/ranaroussi/yfinance)，僅供個人研究，非投資建議
- 新聞快取 TTL 1 小時（`data/{TICKER}_news.json`）；股價快取同天有效
- 更換 Finnhub Key 後需清除舊快取：`rm data/*_news.json`
- VADER 為英文情緒模型，中文標題自動跳過評分（視為中性），不影響英文新聞評分結果
- Finnhub 免費方案限 60 次/分鐘；10 支股票以內通常不會觸發限制
